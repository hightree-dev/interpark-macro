# 인터파크 티켓팅 자동화 프로젝트
인공지능 OCR 기반의 티켓팅 자동화 매크로 프로젝트입니다.

## 📅 프로젝트 히스토리
참고: 본 코드는 2025년에 작성된 원본을 기반으로, 2026년 소스코드 공개를 위해 다음 사항을 수정했습니다:
- 민감 정보(ID/PW) 제거 및 환경변수(`.env`) 처리
- 모듈화 및 프로젝트 구조 재편 (Package 기반)
- 코드 포맷 정리 및 한글 주석/Docstring 추가

## 📋 주요 기능

### 스마트 라벨링 (Labeling Tool)
- **EasyOCR 보조 예측**: 초기 데이터 라벨링 효율 극대화
- **커스텀 모델 교차 검증**: 학습 중인 모델의 예측값 실시간 확인
- **자동 학습 트리거**: 데이터 50개 누적 시 백그라운드 재학습 실행

### 커스텀 OCR 모델 (OCR Engine)
- **CNN-RNN 기반**: CNN(특징 추출) + Bi-GRU(시퀀스 학습) 조합
- **CTC Loss**: 라벨 위치 정보 없이 학습 가능한 End-to-End 방식
- **ONNX 최적화**: 추론 엔진 독립성 및 빠른 추론 속도(~50ms)

### 자동화 매크로 (Automation)
- **브라우저 제어**: Undetected Selenium을 활용한 봇 탐지 우회
- **하이브리드 추론**: OCR 예측값 기반의 캡챠 자동 입력

## 📁 프로젝트 구조
```
interpark-macro/
├── captcha_ocr/          # 🧠 OCR 패키지 (모델 및 학습)
├── labeling_tool/        # 🏷️ 스마트 라벨링 툴
├── automation/           # 🤖 티켓팅 자동화 모듈
├── data_collection/      # 📦 데이터 수집 크롤러
├── weights/              # 모델 가중치 (저장소 미포함)
├── data/                 # 학습 데이터 (저장소 미포함)
├── docs/                 # 작업 흐름도 및 문서
├── main.py               # 메인 실행 스크립트
└── README.md
```

## 🚀 사용법

### 1. 라벨링 및 학습
```bash
# 라벨링 도구 실행
python labeling_tool/gui.py

# 모델 학습 실행
python captcha_ocr/trainer.py
```

### 2. 매크로 실행
```bash
python main.py
```

## ⚙️ 설정
`.env.example` 파일을 복사하여 `.env` 파일을 생성한 후 정보를 입력합니다.
- **계정 정보**: `LOGIN_ID`, `LOGIN_PW`
- **시스템 설정**: 이미지 크기, 학습 캐릭터 셋 등

## 📌 주요 기술 포인트
1. **Active Learning 루프**: 라벨링-학습-예측이 선순환되는 파이프라인 구축
2. **엔드투엔드 모델링**: 별도의 문자 분할 없이 이미지 전체를 시퀀스로 인식
3. **ONNX 런타임**: PyTorch 의존성 없이 가벼운 환경에서 추론 가능
4. **봇 탐지 우회**: 자동화 도구 감지를 최소화한 Selenium 설정 적용

## 💡 프로젝트의 한계와 회고
이 프로젝트는 기술적 완벽함보다는 <b>'문제를 해결해 나가는 즐거움'</b>에 집중했기에, 지금의 시선에서 보면 여러 한계점들이 존재합니다.

1. <b>지표 없는 학습</b>: 당시에는 정교한 평가지표에 대한 개념이 부족했습니다. 그저 학습을 돌려보고 Loss가 떨어지는 것을 확인한 뒤, 실제로 돌려봤을 때 <b>경험적으로 "잘 된다"고 느끼는 수준</b>에서 만족하며 사용했습니다.
2. <b>성능 개선의 한계</b>: 모델의 성능을 체계적으로 업그레이드하거나 최적화하는 방법론을 잘 모르는 상태에서 진행되었습니다. 인공지능 기초 지식만으로 사람이 하기 힘든 일을 자동화했다는 점에 의의를 두었습니다.
3. <b>이미 존재하는 바퀴 (Reinverting the Wheel)</b>: 당시에는 허깅페이스(Hugging Face)처럼 이미 잘 만들어진 모델이나 라이브러리가 존재한다는 사실을 몰랐습니다. 결과적으로는 이미 세상에 있는 바퀴를 다시 만드는 비효율적인 길을 걸었을지도 모릅니다.
4. <b>결론</b>: 비록 돌아가는 길이었고 부족한 점도 많았지만, <b>인공지능이라는 도구를 이용해 내가 가진 실무적 문제를 직접 풀 수 있다는 사실을 온몸으로 경험</b>했다는 것만으로도 저에게는 가장 소중한 첫 번째 AI 프로젝트로 기억되고 있습니다.
